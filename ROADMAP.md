## ✅ Сделано

### Скачивание (ядро)
- yt-dlp + ffmpeg, бэкенд FastAPI, фронтенд — один `backend/static/index.html`.
- Плейлисты → выбор серий галочками; качество (лучшее/1080/720/480/mp3).
- Прогресс в реальном времени; мини-игра «Тетрис» как экран загрузки.
- Автоудаление файлов через 3 часа; размер файла; список файлов на сервере.
- Остановка и удаление задач; необязательный пароль на вход.

### Транскрибация видео в текст
- Сначала готовые субтитры через yt-dlp (мгновенно, **оригинальный язык** —
  приоритет `info["language"]`, затем ru/en/любой), одним запросом.
- Если субтитров нет — локальный **Whisper** (faster-whisper, CPU, бесплатно).
- Лёгкий аудио-формат (`bestaudio/worstaudio/worst`) — не качаем весь видеофайл.
- Ускорение: `beam_size=1` + `vad_filter` (пропуск тишины).
- Понятный статус по стадиям («Скачиваю аудио…» → «Загружаю модель…» →
  «Распознаю NN%») + счётчик времени; кнопка «Остановить».
- Результат — текст на странице + кнопка «Копировать». Модель через `WHISPER_MODEL`.

### Безопасность (перед выводом в интернет)
- **Пароль `DOWNLOADER_PASSWORD`** — основная защита (включён на проде).
- SSRF-guard: только http/https на публичные адреса (блок localhost, приватных
  сетей, облачных метаданных) на `/info`, `/download`, `/transcribe`.
- Потолок одновременных задач (`JOB_CEILING`/`TJOB_CEILING`) → 429 при флуде.
- Whisper сериализован семафором (`MAX_ACTIVE_TRANSCRIBE`), лимит длины ролика
  (`WHISPER_MAX_MINUTES`); необязательный лимит размера файла (`DOWNLOAD_MAX_GB`).

### Интерфейс
- Редизайн в стиле клейморфизм: приглушённый градиент (teal→персик),
  коралловый акцент, маджента-свечение на фокусе, шрифты Space Grotesk + Inter.

### Инфраструктура / деплой
- Работает на **VM-apps-vpn** (egress через VPN → YouTube доступен).
- Виден в интернете: `http://91.186.220.103:8000`
  (Интернет → VPS nginx :8000 → frps → frp-туннель → VM-apps-vpn :8000).
- Обновление: `cd ~/video-downloader && git pull && sudo systemctl restart video-downloader`
  (для правок только фронтенда рестарт не нужен — хватает git pull + Ctrl+F5).

## 🚧 Следующий этап — просмотр видео + загрузка через torrent

Цель: вставил magnet/`.torrent` → **смотришь видео в браузере, пока оно качается**
по торренту, и файл остаётся на сервере.

Набросок плана (детали — при старте этапа):
1. **Торрент-клиент на сервере.** Кандидаты: `libtorrent` (python-биндинги) или
   `aria2` + ffmpeg. Нужен режим **последовательной загрузки** (sequential
   download), чтобы можно было стримить с начала файла, не дожидаясь конца.
2. **API:** `POST /api/torrent` (magnet/torrent → job), статус загрузки (как у
   download), список файлов внутри торрента → выбор нужного видео.
3. **Стриминг в браузер:** отдача файла с поддержкой HTTP Range, плеер
   `<video>` на странице (играет по мере докачки первых кусков).
4. **Кодеки:** mp4/H.264 играет нативно; для mkv/непод­держиваемых — опционально
   ремукс/транскод через ffmpeg (тяжелее, решим по необходимости).
5. **Сеть/безопасность:** торрент-трафик идёт через VPN (приватность) — проверить
   связность пиров/трекеров через туннель; лимиты места на диске и числа задач;
   доступ под тем же паролем; юридическая оговорка (личный просмотр легального).
